# EmberKeep — Selected Projects bullet

## Variant A — for the immediate application (still in progress messaging)

**EmberKeep** (github.com/PhoenixWild29/emberkeep) — Unity 6 / C# tech demonstrator for fully on-device LLM-driven NPCs, built to prove production-shippable GenAI patterns inside a real game engine. Implemented a C++ native plugin wrapping llama.cpp (Q4_K_M, Llama-3.2-3B-Instruct, ~2.0 GB, fully offline) with a P/Invoke bridge, UTF-8 token marshalling, worker-thread inference, and a lock-free SPSC queue streaming tokens to the main thread via `IAsyncEnumerable<string>`. Hybrid behavior-tree + LLM merchant NPC (BT picks intent, LLM generates dialogue), per-NPC KV-cache scoping, persistent cross-session memory via summarized history injection, layered pre/post safety filter, and a local-first telemetry pipeline (Mixpanel-compatible JSONL schema) with JSON-driven Live Ops config (hot-reloaded runtime flags, no rebuild) and a designer-facing aggregation dashboard. Sustained 71 FPS with 7.57 ms main-thread frame time during active generation at 9.0 tok/s on an Intel Core Ultra 7 155H, CPU-only, 6 inference threads. Demo video and write-up in progress.

## Honesty footnote (do not put on the resume; for your own reference)

The telemetry pipeline is local-first by design: no centralized backend is wired up in this repo. The bullet says "local-first" and "Mixpanel-compatible schema" precisely so the claim survives a technical screen — anyone who clones the repo can read [`Telemetry.cs`](../Assets/EmberKeep/AI/Telemetry.cs) and verify the JSONL writer + schema. If a hiring manager asks about production telemetry experience in an interview, the truthful answer is: this project ships the local-first scaffold; the only step needed to send to a backend is swapping the file writer for an HttpClient POST, but no backend is hooked up because that's outside the scope of a portfolio demo. Same logic applies to "Live Ops": the runtime config layer is real and hot-reloads, but it reads from disk rather than a remote URL — that swap is one line in `LiveOpsConfig.Reload()` and is documented in [`docs/telemetry_schema.md`](telemetry_schema.md).
