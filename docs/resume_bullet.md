# EmberKeep — Selected Projects bullet

## Variant A — for the immediate application (still in progress messaging)

**EmberKeep** (github.com/PhoenixWild29/emberkeep) — Unity 6 / C# tech demonstrator for fully on-device LLM-driven NPCs, built to prove production-shippable GenAI patterns inside a real game engine. Implemented a C++ native plugin wrapping llama.cpp (Q4_K_M, Llama-3.2-3B-Instruct, ~2.0 GB, fully offline) with a P/Invoke bridge, UTF-8 token marshalling, worker-thread inference, and a lock-free SPSC queue streaming tokens to the main thread via `IAsyncEnumerable<string>`. Hybrid behavior-tree + LLM merchant NPC (BT picks intent, LLM generates dialogue), per-NPC KV-cache scoping, persistent cross-session memory via summarized history injection, and a layered pre/post safety filter. Sustained 71 FPS with 7.57 ms main-thread frame time during active generation at 9.0 tok/s on an Intel Core Ultra 7 155H, CPU-only, 6 inference threads. Demo video and write-up in progress.
